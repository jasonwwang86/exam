# Figma Prompts For Admin Console Redesign

## Purpose

本文件用于驱动 `Figma` 插件为 `exam-web` 生成全新的后台管理端方向稿。目标不是延续旧原型，而是在保持功能不变的前提下，彻底重做视觉与信息架构。

## Shared Context

- Project: `exam`
- Frontend scope: `exam-web`
- Style target: 企业级后台管理端，强调秩序感、专业度、可读性和高频操作效率
- Constraint: 功能保持不变，但允许完全推翻现有页面结构
- Avoid: 默认 Ant Design 卡片堆砌感、弱层级、单一蓝白渐变、过度营销化 Hero

## Visual Reference Summary

结合新增参考图，本次更偏向以下视觉特征：

- 浅色、扁平、克制的业务后台
- 左侧固定导航，支持树状业务分组
- 顶部保留标签页或面包屑式工作区切换
- 主内容区以白色工作面板为主，背景弱化
- 查询区、工具栏、表格区上下分层清晰
- 图表、概览、快捷入口都更偏“实用业务系统”，而不是展示型大屏
- 首页更像考试平台工作台，而不是数据海报
- 整体接近成熟考试管理系统、政企内管系统、培训考试平台风格

不希望出现的方向：

- 过深色的中控大屏感
- 过强的玻璃拟态或渐变装饰
- 过多独立卡片拼贴
- 偏官网、偏营销、偏品牌宣传页的首页气质

## Fixed Functional Scope

### Login Page

- 保留用户名输入框
- 保留密码输入框
- 保留错误提示区域
- 保留登录按钮

### Post-login Shell

- 保留侧边导航
- 保留顶部用户信息
- 保留退出登录入口
- 保留内容主区域

### Dashboard

- 保留“管理首页”的统一工作台定位
- 保留系统概览和模块概览信息承载能力

### Examinee Management

- 保留筛选区
- 保留列表表格
- 保留新增、编辑、删除、状态切换
- 保留导入、导出
- 保留权限驱动的操作显隐

## Prompt A

```md
使用 Figma 插件为 exam-web 设计一套深色企业中控台方向的后台管理端方案，功能保持不变，但允许完全推翻现有布局。

请输出以下页面：
1. 管理员登录页
2. 登录后的统一后台框架
3. 管理首页
4. 考生管理页

必须保留的功能：
- 登录页保留用户名、密码、错误提示、登录按钮
- 后台框架保留侧边导航、顶部用户信息、退出登录
- 首页保留管理概览能力
- 考生管理页保留筛选、表格、新增、编辑、删除、状态切换、导入、导出

设计要求：
- 风格定位为深色企业中控台，不要炫技型 3D，也不要营销官网感
- 强调运营驾驶舱的秩序感、专业感和高信息密度
- 首页不要只是统计卡片拼贴，要更像真实工作台
- 考生管理页要突出筛选、批量操作入口和表格可读性
- 登录页与主界面要属于同一产品体系
- 允许彻底重做布局，但不能改变功能语义
- 桌面端优先，同时考虑移动端收敛方式
```

## Review Criteria

- 首屏辨识度是否足够强
- 登录页与主界面是否属于同一体系
- 信息密度是否适合后台产品
- 筛选区、工具区、表格区层级是否清楚
- 是否摆脱了默认卡片堆砌感
- 后续新增模块能否沿用这套骨架

## Recommendation

- 先同时生成两套方向稿，再做选型
- 定稿后再进入 OpenSpec 和代码实现
- 不建议在定稿前直接修改 `exam-web` 页面结构
