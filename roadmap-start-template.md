# 新模块会话启动模板

## 1. 使用目的

本模板用于在开启一个“新的模块开发会话”时，快速向 AI 或协作者说明当前模块的边界、目标和限制，避免上下文漂移、范围失控和跨模块混入。

建议遵循以下原则：

- 一个模块对应一个独立会话
- 一个会话优先只服务一个 OpenSpec change
- 新会话开头只说明“当前模块范围”，不要重复粘贴整份路线图
- 如果模块边界不清楚，先 explore；如果边界清楚，直接 propose

## 2. 通用启动模板

可直接复制下面内容到新会话中使用：

```text
当前模块：<模块名称>

参考文档：
- roadmap.md
- AGENTS.md
- openspec/config.yaml

本次目标：
- <本轮要达成的业务目标 1>
- <本轮要达成的业务目标 2>

本次范围：
- <本次包含的功能点 1>
- <本次包含的功能点 2>
- <本次包含的功能点 3>

本次不包含：
- <明确不做的内容 1>
- <明确不做的内容 2>
- <明确不做的内容 3>

依赖前置模块：
- <依赖模块 1>
- <依赖模块 2>

涉及系统：
- exam-web: <是否涉及，以及涉及内容>
- exam-service: <是否涉及，以及涉及内容>

请按以下方式推进：
- 先基于当前模块边界判断是否需要 OpenSpec explore
- 如果边界已经清楚，直接创建本模块的 OpenSpec proposal
- 不要扩展到其他模块
- proposal、design、tasks 中请明确标注本模块的包含范围与不包含范围
```

## 3. 精简版启动模板

如果你只是想快速开工，可以使用更短的版本：

```text
当前模块：<模块名称>
参考文档：roadmap.md
本次范围：<一句话描述本次要做什么>
本次不包含：<一句话描述本次明确不做什么>
请先按该范围创建 OpenSpec proposal，不要扩展到其他模块。
```

## 4. 推荐填写方式

### 4.1 当前模块

只填写当前准备推进的单一模块，例如：

- 题库管理
- 试卷管理
- 考试计划
- 在线答题

不要写成：

- 完整考试系统
- 管理端全部功能
- 考生端全部流程

### 4.2 本次目标

目标写“这轮交付后能做什么”，例如：

- 管理员能够维护试题
- 管理员能够创建考试计划
- 考生能够完成在线答题

### 4.3 本次范围

范围写“本轮包含的功能点”，控制在 3 到 6 条最合适。  
这些内容应当能够共同构成一个可独立验收的模块。

### 4.4 本次不包含

这是最关键的一部分。  
必须明确排除掉容易被带进来的相邻模块，例如：

- 本次不包含组卷
- 本次不包含考试计划
- 本次不包含成绩单

### 4.5 依赖前置模块

如果当前模块依赖前置模块，建议显式写出来，便于 AI 理解上下游关系，例如：

- 依赖题库管理
- 依赖考生管理
- 依赖考试计划

## 5. 适合直接 propose 的场景

下面这些情况下，通常可以不先 explore，直接基于模板进入 propose：

- 模块边界已经很清楚
- 包含范围和不包含范围都已明确
- 依赖关系清晰
- 你已经能说清楚“这一轮只做什么”

## 6. 适合先 explore 的场景

下面这些情况下，建议先 explore 再决定 proposal 边界：

- 当前模块是否需要再拆成两个 change 还不确定
- 本轮是否应该包含相邻模块还不确定
- 前后端接口边界还不稳定
- 数据结构和流程边界还不清楚

## 7. 按模块输出示例

下面示例直接对应 `roadmap.md` 中的模块，你后续可以直接复制某一段作为新会话起点。

### 7.1 管理端基础权限与登录

```text
当前模块：管理端基础权限与登录

参考文档：
- roadmap.md
- AGENTS.md
- openspec/config.yaml

本次目标：
- 管理员能够登录系统
- 系统具备基础角色与权限控制能力

本次范围：
- 登录页
- 登录接口
- Token 或 Session 机制
- 基础用户、角色、权限模型
- 菜单权限或接口权限控制

本次不包含：
- 考生管理
- 题库管理
- 大屏可视化

依赖前置模块：
- 无

涉及系统：
- exam-web: 管理端登录页、基础鉴权与权限路由控制
- exam-service: 认证接口、用户角色权限模型、鉴权逻辑

请按以下方式推进：
- 使用 brainstorming 做一次边界核对，先判断是否需要 OpenSpec explore
- 如果边界已经清楚，直接创建管理端基础权限与登录模块的 OpenSpec proposal
- 不要扩展到考生管理或题库管理
- proposal、design、tasks 中请明确标注本模块的包含范围与不包含范围
```

## 8. 建议的使用方式

后续每次开启新模块会话时，建议操作顺序如下：

1. 先查看 `roadmap.md` 中对应模块的建议范围
2. 复制本模板到新会话
3. 只替换当前模块相关内容
4. 明确写出“本次不包含”
5. 让 AI 判断先 explore 还是直接 propose

## 9. 一句话原则

新模块会话的重点不是“重新介绍整个系统”，而是“明确告诉当前会话，这一轮只做什么，不做什么”。
