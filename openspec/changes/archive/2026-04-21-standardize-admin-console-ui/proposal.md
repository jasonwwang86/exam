## Why

当前 `exam-web` 登录后的统一主页面和业务页面仍以自定义 CSS 卡片风格为主，整体观感和信息组织方式与常见企业级管理台存在明显差距，也尚未形成可复用的共享页面规范。随着后续管理模块继续接入，如果继续沿用当前做法，会放大页面风格不一致、共享布局难复用和维护成本上升的问题。

## What Changes

- 将 `exam-web` 登录成功后的统一主页面升级为企业级管理台风格的共享壳层，统一顶部信息区、侧边导航区、内容承载区和当前菜单高亮规则。
- 将 `exam-web` 的管理员登录页升级为与主界面一致的主题入口页，统一品牌标识、配色、圆角、阴影和表单控件风格，并采用居中单卡片结构，不再保留左侧介绍区域，同时保持现有登录流程和校验行为不变。
- 在 `exam-web/src/shared` 中沉淀管理端页面的统一展示模式，包括页面标题区、筛选区、操作区、内容卡片区以及空态、加载态、错误态的表现约定。
- 以 Ant Design 作为本次管理端样式升级的基础组件体系，重构管理首页与考生管理页面的视觉层级、间距和常用交互控件，但保持现有权限控制和 REST 调用链路不变。
- 收敛模块页的顶部呈现，移除介绍型 Hero 抬头，仅保留必要标题、操作区与正文内容；同时将考生管理中的新增、编辑、导入切换为弹框交互，其中导入入口由顶部 `新增` 下拉操作承载。
- 本次变更不修改 `exam-service`、不新增或变更 REST 端点、不调整请求/响应 DTO，也不涉及数据库表结构和 MyBatis-Plus Mapper。

## Capabilities

### New Capabilities
- `admin-console-ui`: 定义 `exam-web` 登录后统一管理台壳层与标准化管理页面结构，供首页和后续业务模块复用。

### Modified Capabilities
- `admin-authentication`: 调整管理员登录页在 `exam-web` 中的视觉呈现，使未登录入口与主界面保持同一主题语言。
- `examinee-management`: 调整考生管理页面在统一管理台中的呈现要求，使其遵循标准化查询区、操作区和数据区布局。

## Impact

- `exam-web`：影响 `src/shared/layouts`、`src/shared/styles`、登录页、管理首页页面以及考生管理页面，新增或调整共享页面容器和管理台主题配置。
- 前端依赖：复用当前已安装的 `antd`，将其从“已安装但未系统使用”调整为“管理端 UI 基础组件体系”。
- `exam-service`：REST 端点保持不变，继续复用现有 `/api/admin/auth/me`、`POST /api/admin/auth/logout`、`GET /api/admin/examinees` 及相关增删改导入导出接口。
- 数据层：无 MySQL 表结构变更，无 Mapper、DTO、实体类调整。
