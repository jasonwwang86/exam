## MODIFIED Requirements

### Requirement: `exam-web` 应提供由 `exam-service` 支撑的考生管理模块入口
系统 MUST 在 `exam-web` 中为有权限的管理员提供考生管理能力，并由受保护的 `exam-service` 接口支撑。考生管理视图 MUST 支持关键字搜索、状态筛选、分页与列表操作，且不得向浏览器直接暴露持久化实体。考生管理页面 MUST 在统一管理台壳层中，以紧凑标题区、查询条件区、操作区和数据表格区呈现，不再展示介绍型模块抬头，并与其他管理模块保持一致的状态反馈样式。

#### Scenario: 管理员按条件查询考生
- **WHEN** 具备考生查询权限的管理员在 `exam-web` 打开考生管理页面并提交关键字或状态筛选条件
- **THEN** 考生管理页面必须展示统一的查询条件区、操作区和数据区
- **THEN** `exam-web` 必须调用 `GET /api/admin/examinees`
- **THEN** `exam-service` 必须返回管理端展示所需的分页结果，至少包含考生 ID、考生编号、姓名、手机号、状态和更新时间

#### Scenario: 管理员通过 `新增` 聚合入口发起新增或导入
- **WHEN** 具备考生维护权限的管理员在考生管理页面点击顶部 `新增` 操作入口
- **THEN** `exam-web` 必须展示至少包含 `新增考生` 和 `导入文件` 的下拉操作项
- **THEN** 选择 `新增考生` 后必须以弹框方式承载考生表单
- **THEN** 选择 `导入文件` 后必须以弹框方式承载文件选择与导入提交

#### Scenario: 管理员以弹框方式编辑考生
- **WHEN** 具备考生编辑权限的管理员在列表中点击 `编辑`
- **THEN** `exam-web` 必须以弹框方式展示当前考生资料
- **THEN** 提交后必须继续调用既有更新接口并刷新当前列表

#### Scenario: 无权限管理员不能进入考生模块
- **WHEN** 已登录管理员缺少考生管理菜单权限或相关 API 权限
- **THEN** `exam-web` 必须隐藏或阻止进入考生管理入口及相关操作按钮
- **THEN** `exam-service` 必须拒绝对应的受保护考生接口请求

#### Scenario: 考生列表为空时展示统一空态
- **WHEN** 具备考生查询权限的管理员进入考生管理页面，且当前筛选条件下没有任何记录
- **THEN** `exam-web` 必须在统一数据区中展示明确的空态提示
- **THEN** 页面仍必须保留查询条件区和操作区，便于管理员继续筛选或新增数据
