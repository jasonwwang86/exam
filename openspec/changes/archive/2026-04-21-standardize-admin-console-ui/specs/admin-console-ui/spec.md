## ADDED Requirements

### Requirement: `exam-web` 应提供企业级管理台风格的统一主页面壳层
系统 MUST 在管理员登录成功后，为 `exam-web` 中所有已接入的管理模块提供一致的企业级管理台壳层。该壳层 MUST 至少包含顶部信息区、侧边导航区、当前模块高亮状态以及稳定的主内容承载区域，并允许后续模块沿用同一套布局接入。

#### Scenario: 已登录管理员进入受保护页面
- **WHEN** 已登录且具备可见菜单的管理员进入 `exam-web` 的任一受保护管理页面
- **THEN** 页面必须展示统一的顶部区域、侧边菜单区域和主内容区域
- **THEN** 当前路由对应的菜单项必须处于高亮状态，且主内容不得脱离统一壳层单独渲染

#### Scenario: 当前账号没有可见菜单
- **WHEN** 已登录管理员进入统一主页面，但当前账号没有可见业务菜单
- **THEN** `exam-web` 必须仍展示统一主页面壳层
- **THEN** 页面必须给出明确的无菜单提示，而不是退化为无导航的裸页面

### Requirement: `exam-web` 应为管理首页和列表型业务页提供统一页面结构
系统 MUST 为 `exam-web` 的管理首页和列表型业务页面提供一致的页面结构，包括页面标题区、说明信息区、筛选区、操作区、数据内容区以及统一的加载态、空态和错误态承载方式，以便后续模块复用同一管理台模式接入。

#### Scenario: 管理首页沿用统一内容结构
- **WHEN** 管理员进入管理首页
- **THEN** 首页必须在统一主页面壳层内展示页面标题、概要说明和信息卡片区域
- **THEN** 首页内容宽度、间距层级和卡片承载方式必须与其他管理模块保持一致

#### Scenario: 列表型业务页沿用统一内容结构
- **WHEN** 管理员进入任一列表型业务页
- **THEN** 页面必须以统一的标题区、筛选区、操作区和数据区呈现
- **THEN** 加载态、空态和错误态必须在统一的数据内容区域内展示，而不是散落在页面不同位置
