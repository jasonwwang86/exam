## MODIFIED Requirements

### Requirement: `exam-web` 应为管理首页和列表型业务页提供统一页面结构
系统 MUST 为 `exam-web` 的管理首页和列表型业务页面提供一致的业务后台页面结构。该结构 MUST 贴近成熟考试业务系统的工作台模式，以白底工作区、清晰的区块边界和稳定的信息层级承载页面内容，而不是依赖默认卡片堆砌。首页 MUST 以工作台结构展示个人信息、本月数据和常用功能三个区块；列表型业务页 MUST 以筛选区、工具栏和数据表格区的三段式结构呈现，并在统一数据区中承载加载态、空态和错误态。

#### Scenario: 管理首页沿用收敛后的统一工作台结构
- **WHEN** 管理员进入管理首页
- **THEN** 首页必须在统一主页面壳层内展示个人信息区、本月数据区和常用功能区
- **THEN** 首页不得继续展示与当前系统能力无关的静态趋势图、系统概览或介绍型占位内容

#### Scenario: 首页个人信息与常用功能基于当前登录态展示
- **WHEN** 管理员在 `exam-web` 中打开 `/dashboard`
- **THEN** 个人信息区必须基于当前登录账号的真实字段展示姓名、账号、角色或等效身份摘要
- **THEN** 常用功能区必须基于当前账号已有可见菜单展示可进入的模块入口，而不是展示固定硬编码功能列表

#### Scenario: 列表型业务页沿用三段式业务结构
- **WHEN** 管理员进入任一列表型业务页
- **THEN** 页面必须以筛选区、工具栏和数据表格区的三段式结构呈现
- **THEN** 加载态、空态和错误态必须在统一的数据表格承载区中展示，而不是散落在页面不同位置

#### Scenario: 页面风格保持浅色扁平业务后台特征
- **WHEN** 管理员在 `exam-web` 中浏览管理首页或任一列表型业务页
- **THEN** 页面必须使用浅色、扁平、以白底工作区为主的后台视觉语言
- **THEN** 页面不得依赖大面积玻璃拟态、强装饰渐变或过度卡片化布局来承载主要业务内容
