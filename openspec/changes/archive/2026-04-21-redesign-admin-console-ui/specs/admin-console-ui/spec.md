## MODIFIED Requirements

### Requirement: `exam-web` 应提供企业级管理台风格的统一主页面壳层
系统 MUST 在管理员登录成功后，为 `exam-web` 中所有已接入的管理模块提供一致的企业级考试业务管理台壳层。该壳层 MUST 采用浅色、扁平、以业务工作区为中心的布局结构，至少包含左侧导航区、顶部工作区头部、当前模块高亮状态以及稳定的白底主内容承载区域，并允许后续模块沿用同一套布局接入。

#### Scenario: 已登录管理员进入受保护页面
- **WHEN** 已登录且具备可见菜单的管理员进入 `exam-web` 的任一受保护管理页面
- **THEN** 页面必须展示左侧导航区、顶部工作区头部和白底主内容区域
- **THEN** 当前路由对应的菜单项必须处于高亮状态，且主内容不得脱离统一壳层单独渲染

#### Scenario: 壳层展示当前工作区上下文
- **WHEN** 管理员在 `exam-web` 中进入首页、考生管理页或后续其他已接入模块
- **THEN** 顶部区域必须展示当前工作区的上下文信息
- **THEN** 该上下文展示必须采用接近工作区标签页或工作区切换条的轻量结构，而不是只保留纯文本标题

#### Scenario: 当前账号没有可见菜单
- **WHEN** 已登录管理员进入统一主页面，但当前账号没有可见业务菜单
- **THEN** `exam-web` 必须仍展示统一主页面壳层
- **THEN** 页面必须给出明确的无菜单提示，而不是退化为无导航的裸页面

### Requirement: `exam-web` 应为管理首页和列表型业务页提供统一页面结构
系统 MUST 为 `exam-web` 的管理首页和列表型业务页面提供一致的业务后台页面结构。该结构 MUST 贴近成熟考试业务系统的工作台模式，以白底工作区、清晰的区块边界和稳定的信息层级承载页面内容，而不是依赖默认卡片堆砌。首页 MUST 以工作台结构展示个人信息、统计概览、常用功能和图表概览；列表型业务页 MUST 以筛选区、工具栏和数据表格区的三段式结构呈现，并在统一数据区中承载加载态、空态和错误态。

#### Scenario: 管理首页沿用统一工作台结构
- **WHEN** 管理员进入管理首页
- **THEN** 首页必须在统一主页面壳层内展示个人信息区、统计概览区、常用功能区和图表概览区
- **THEN** 首页必须作为真实业务工作台呈现，而不是只展示介绍文案和简单信息卡片拼贴

#### Scenario: 列表型业务页沿用三段式业务结构
- **WHEN** 管理员进入任一列表型业务页
- **THEN** 页面必须以筛选区、工具栏和数据表格区的三段式结构呈现
- **THEN** 加载态、空态和错误态必须在统一的数据表格承载区中展示，而不是散落在页面不同位置

#### Scenario: 页面风格保持浅色扁平业务后台特征
- **WHEN** 管理员在 `exam-web` 中浏览管理首页或任一列表型业务页
- **THEN** 页面必须使用浅色、扁平、以白底工作区为主的后台视觉语言
- **THEN** 页面不得依赖大面积玻璃拟态、强装饰渐变或过度卡片化布局来承载主要业务内容
